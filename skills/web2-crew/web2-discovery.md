# Skill: web2-discovery

## Purpose
Run Pre-Start, Phase 1 (Pre-Ideation), and Phase 2 (Ideation) to produce a complete project foundation package.

## Triggers
- "start web2 discovery"
- "run pre-start"
- "onboard this client"
- "create project scope"
- "kickoff web2 project"

## Required Inputs
- Client/business name
- Offerings (products/services)
- Target audience
- Primary goals (top 3)
- Timeline target
- Budget range (optional but recommended)
- Stakeholders and decision maker
- Existing brand assets (if any)
- Competitors (3-5)

## Optional Inputs
- Existing website URL
- Analytics baseline
- CRM/marketing tools
- Compliance constraints (GDPR/CCPA, ADA needs)

## Workflow
1. Intake and qualification
2. Discovery questionnaire completion
3. KPI and success metrics definition
4. Scope boundary and assumptions
5. Milestone timeline draft
6. Stakeholder map + communication cadence
7. Brand positioning draft
8. Ideation workshop agenda + synthesis
9. Risk register v1
10. Handoff package creation

## Deliverables
- `01-client-profile.md`
- `02-project-scope.md`
- `03-kpi-framework.md`
- `04-timeline-milestones.md`
- `05-stakeholder-comms-plan.md`
- `06-brand-positioning.md`
- `07-ideation-summary.md`
- `08-risk-register.md`
- `handoff-discovery-to-cue.md`

## Quality Gates
- Scope includes in/out boundaries
- KPIs are measurable and time-bound
- Timeline has owners + target dates
- Stakeholder approvals captured
- Top risks have mitigation actions
- Positioning statement validated by client

## Handoff To
- `web2-cue-strategy`
- `web2-pm-orchestrator`

## Failure Modes + Recovery
- Missing client input -> issue input checklist + proceed with placeholders clearly marked
- Scope ambiguity -> create decision log with recommended default
- Conflicting stakeholder feedback -> escalate to PM decision checkpoint
- Unrealistic timeline -> produce phased MVP + phase 2 roadmap

## Completion Criteria
Discovery package approved by client and PM, with no unresolved critical blockers.
