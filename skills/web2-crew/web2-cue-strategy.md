# Skill: web2-cue-strategy

## Purpose
Execute Phase 3 (Media Collection) and Phase 4 (C.U.E Strategy) to produce content, UX, and SEO-ready planning artifacts.

## Triggers
- "run cue strategy"
- "start content ux seo planning"
- "prepare IA and content map"
- "build web2 content strategy"

## Required Inputs
- Approved discovery package
- Brand guide (or brand direction)
- Audience segments/personas
- Core offers/services
- Priority pages and conversion goals
- Competitor references

## Optional Inputs
- Existing sitemap/IA
- Existing content library
- Keyword seed list
- Tone of voice examples

## Workflow
1. Audit available assets and information gaps
2. Define page-level goals and primary CTAs
3. Build sitemap and information architecture
4. Map user journeys by funnel stage
5. Perform keyword clustering by intent
6. Create content map (page -> message -> keyword -> CTA)
7. Draft wireframe guidance and section hierarchy
8. Define on-page SEO requirements
9. Produce editorial calendar (initial 30-60 days)
10. Package handoff for WMD and CARE

## Deliverables
- `09-asset-audit.md`
- `10-sitemap-ia.md`
- `11-user-journey-map.md`
- `12-keyword-clusters.md`
- `13-content-map.md`
- `14-wireframe-guidance.md`
- `15-on-page-seo-spec.md`
- `16-editorial-calendar.md`
- `handoff-cue-to-wmd.md`

## Quality Gates
- Every priority page has a clear conversion goal
- Sitemap aligns to business goals and personas
- Keywords mapped by intent (awareness/consideration/decision)
- Content map includes CTA and internal link targets
- Wireframe guidance includes mobile-first hierarchy
- SEO spec includes title/meta/heading/url/alt-text rules

## Handoff To
- `web2-build-qa`
- `web2-pm-orchestrator`

## Failure Modes + Recovery
- Weak/missing brand inputs -> produce provisional messaging framework for approval
- Content volume exceeds timeline -> prioritize revenue-critical pages first
- Keyword conflicts across pages -> enforce one primary intent per page
- Stakeholder disagreement on IA -> run decision workshop with scored options

## Completion Criteria
Client and PM approve sitemap, content map, and SEO specification with no critical open dependencies.
